# Skill 使用指南编写规范

> 当用户要求编写 Skill 的"使用说明"/"使用指南"/"用户手册"时，必须遵循以下规范。

## 核心原则

- **面向普通用户**：读者是在 Antigravity 对话框中操作的普通用户，不是开发者
- **零技术门槛**：禁止出现 `python "xxx/scripts/xxx.py" --arg` 等命令行写法
- **对话框驱动**：所有操作以"在对话框中输入什么"的方式呈现
- **场景化示例**：用"你的需求 → 在对话框输入"的对照表，而非参数说明

## 禁止事项

- ❌ 暴露脚本路径（如 `<SKILL_DIR>\scripts\xxx.py`）
- ❌ 展示 Python/PowerShell 命令行（如 `python xxx init --server ...`）
- ❌ 列出 CLI 参数表（如 `--server`, `--database`, `--auth`）
- ❌ 使用开发者术语解释用户操作（如"执行 init 子命令"）

## 必须使用的写法

- ✅ `在对话框输入：对 oa 数据库做代码审查`
- ✅ `直接说：帮我检查一下数据库有什么问题`
- ✅ `告诉 AI：继续上次的优化工作`
- ✅ 用表格呈现"你的需求 → 在对话框输入"

## 固定章节结构（12 章）

编写 Skill 使用指南时，按以下结构组织（根据实际情况可省略不相关章节）：

1. **功能概述** — 这个工具能做什么（模块表格，一目了然）
2. **快速开始** — 一个最简单的示例，3 行以内让用户知道怎么用
3. **指令速查表** — 启动 / 查看进度 / 继续 / 报告，用 carousel 或表格
4. **完整操作流程（图解）** — Mermaid 流程图，可视化全流程
5. **分步详解** — 每步包含"在对话框输入什么"+"AI 会做什么"+"你会看到什么"
6. **跨天工作（断点续传）** — 如适用
7. **高级用法** — 如 Auto 模式等进阶功能
8. **各模块详解** — 每个模块的功能、检查内容、触发语句
9. **产出文件说明** — 表格（路径 / 内容 / 你需要关注吗？）
10. **安全保障** — 安全机制说明
11. **常见问题 FAQ** — 5-10 个 Q&A
12. **快速参考卡片** — 一句话启动表 + 推荐顺序 + 关键文件位置

## 格式要求

- 使用 GitHub Alerts（`> [!TIP]`、`> [!IMPORTANT]`、`> [!CAUTION]`）突出关键信息
- 使用 carousel 展示多场景对照
- 使用 Mermaid 图可视化流程
- 每个模块的"怎么触发"给出 1-2 句自然语言对话框示例
- FAQ 用 `### ❓ 问题` 格式，回答简洁非技术化
