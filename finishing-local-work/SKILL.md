# 完成本地工作

## 概述
本 skill 用于在本地开发工作完成后，先验证结果，再给出清晰的收尾选项，并安全清理隔离目录。

**核心原则：** 先验证结果 → 再呈现本地收尾选项 → 按选择执行 → 最后有意识地清理。

**开始时说明：** "我正在使用 finishing-local-work skill 来完成本地工作收尾。"

## 流程
### 第 1 步：验证结果
在给出收尾选项之前，先确认当前结果处于可接受状态。

例如：
- 运行相关测试或验证命令
- 确认生成产物已经成功产出
- 确认必需文件存在且可读取
- 确认输出目录中包含预期文件

**如果验证失败：**
```text
验证失败，必须先修复后才能收尾：

[Show failures]

在结果被验证通过之前，不能继续收尾流程。
```

停止，不进入第 2 步。

**如果验证通过：** 继续第 2 步。

### 第 2 步：识别结果类型
判断这次任务最终产出了什么：
- 源文件修改
- 仅生成输出
- 同时包含源文件修改和生成输出
- 一个临时实验工作区

根据结果类型，选择最安全的收尾方式。

### 第 3 步：呈现选项
严格只给出以下 4 个选项：

```text
实现已经完成。你希望我如何处理？

1. 将已验证的修改应用回源目录
2. 原样保留当前隔离工作区，后续再处理
3. 只保留生成产物
4. 丢弃这次工作

请选择一项。
```

**不要额外解释**，除非用户主动追问。

### 第 4 步：执行选择
#### 选项 1：应用回源目录
适用于隔离工作区中的结果应成为正式结果。

执行动作：
- 将确认无误的源码修改复制或同步回源目录
- 只保留本次真正需要的文件
- 应用完成后重新运行最相关的一次验证
- 如果验证仍然通过，再报告成功

#### 选项 2：原样保留工作区
报告：

```text
已保留隔离工作区：<path>
没有删除任何文件。
```

不要清理该隔离工作区。

#### 选项 3：只保留生成产物
执行动作：
- 保留需要的报告、导出文件或生成产物
- 如有需要，把它们移动或复制到约定好的输出位置
- 删除不再需要的临时中间文件
- 清楚报告最终输出路径

#### 选项 4：丢弃
**必须先确认：**

```text
这将永久删除临时本地工作区，以及其中所有尚未保留的结果。

请输入 'discard' 确认。
```

等待用户给出精确确认。

确认后：
- 删除临时工作区或临时输出目录树
- 报告删除了哪些内容

### 第 5 步：清理
**对于选项 1、3、4：**
- 删除已经不再需要的临时隔离目录
- 保留用户确认需要保留的源码修改或最终产物
- 报告哪些内容被保留、哪些内容被删除

**对于选项 2：** 保留隔离工作区，不做清理。

## 快速参考
| 选项 | 应用回源目录 | 保留工作区 | 保留产物 | 删除临时工作区 |
|------|--------------|------------|----------|----------------|
| 1. 应用回源目录 | ✓ | - | 可选 | ✓ |
| 2. 原样保留 | - | ✓ | 可选 | - |
| 3. 只保留产物 | - | - | ✓ | ✓ |
| 4. 丢弃 | - | - | - | ✓ |

## 常见错误
### 跳过最终验证
- **问题：** 把未完成或有问题的结果当成已完成
- **修正：** 在给出选项前，一定先验证实际结果

### 临时文件与最终产物混在一起
- **问题：** 很难判断哪些应该保留
- **修正：** 在清理前先区分源码修改、最终产物和临时文件

### 过早清理
- **问题：** 用户还没决定保留什么，工作区就被删掉了
- **修正：** 先给出 4 个选项，再按用户选择执行清理

### 丢弃前不确认
- **问题：** 容易误删有价值的工作结果
- **修正：** 必须要求输入精确的 `discard` 作为确认

## 红线
**绝不要：**
- 未经验证就宣称工作已完成
- 未经确认就删除本地工作结果
- 默认把临时产物混入源目录
- 擅自假设用户一定要把修改应用回源目录

**始终要：**
- 先验证，再给收尾选项
- 严格给出 4 个选项
- 丢弃前获取明确确认
- 清楚报告保留了什么、删除了什么
