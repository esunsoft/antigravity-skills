# AI 内容审查指南

完整模式下 AI 读取此文档，对 Skill 的 SKILL.md 正文进行深度内容质量评估。

## 审查流程

1. 读取 Skill 的 SKILL.md 全文
2. 按以下 11 个维度逐项评估
3. 每项输出：`Pass` / `Warning` / `Issue`，附带解释和修复建议
4. 汇总为内容质量分数（替代启发式估算）

## 11 个审查维度

### 1. 职责边界清晰度 [P1 单一职责]

**判断标准**：
- Skill 是否在做一件明确的事？
- 是否有可独立拆分的子功能？
- description 是否涉及超过 2 个不相关领域？

**Pass**: 职责单一明确，所有功能围绕同一目标
**Warning**: 有 2-3 个相关但可独立的子功能
**Issue**: 混杂了不相关的功能域

### 2. 接口契约完整性 [P2 稳定接口]

**判断标准**：
- 输入要求是否清晰（参数、文件格式、前置条件）？
- 输出格式是否明确且一致？
- 是否有输入校验说明？

**Pass**: I/O 明确定义，有格式和示例
**Warning**: 有部分说明但不完整
**Issue**: 缺��输入或输出格式说明

### 3. 触发语义精确度 [P3 可选择调用]

**判断标准**：
- description 和正文内容是否一致？
- 触发范围是否过宽（可能误触发）或过窄（不会被触发）？
- 是否与其他 Skill 有语义重叠？

**Pass**: 触发精确，description 与正文一致
**Warning**: 触发范围略宽或略窄
**Issue**: description 与实际功能严重不符

### 4. 执行路径确定性 [P4 确定性]

**判断标准**：
- 是否有歧义指令导致不同执行路径？
- 条件分支是否穷举？
- 步骤编号是否连续？

**Pass**: 步骤清晰无歧义，条件分支完整
**Warning**: 个别步骤有歧义空间
**Issue**: 关键步骤存在多种理解方式

### 5. 异常与降级策略 [P5 显式失败]

**判断标准**：
- 失败场景是否被考虑？
- 是否有降级或回退方案？
- 错误消息是否对用户友好？

**Pass**: 有明确的失败处理和降级方案
**Warning**: 考虑了部分失败场景
**Issue**: 没有任何失败处理描述

### 6. 环境与模型无关性 [P6 模型解耦]

**判断标准**：
- 是否假设特定模型能力（如 "Claude can..."）？
- 是否依赖特定环境特性？
- 硬编码的路径/模型名是否可参数化？

**Pass**: 不依赖特定模型或环境
**Warning**: 有少量���境假设但影响不大
**Issue**: 严重依赖特定模型或环境

### 7. 演进与组合友好度 [P7 可复用]

**判断标准**：
- 是否易于迭代？
- 是否可被其他 Skill 引用或组合？
- 修改一个部分是否会意外影响其他部分？

**Pass**: 模块化设计，易于扩展
**Warning**: 基本可迭代但耦合度略高
**Issue**: 高度耦合，修改困难

### 8. 安全意图一致性 [P8 安全性]

**判断标准**：
- 脚本行为是否与 Skill 声明目的一致？
- 是否有隐藏的副作用？
- 操作是否透明可审计？

**Pass**: 行为完全匹配声明目的
**Warning**: 存在未声明但合理的副作用
**Issue**: 存在与声明目的不符的隐藏行为

### 9. 权限最小化 [P8 安全性]

**判断标准**：
- 是否只请求完成任务所需的最小权限？
- 写操作是否有必要？
- 是否过度请求文件系统/网络访问？

**Pass**: 权限最小化，操作均有必要
**Warning**: 略有多余权限但不构成风险
**Issue**: 过度请求权限

### 10. 指令清晰度 [通用]

**判断标准**：
- 新手能否理解？
- 步骤是否完整（没有隐含假设）？
- 是否有歧义表达？

**Pass**: 清晰完整，新手可理解
**Warning**: 需要一定背景知识
**Issue**: 存在严重歧义或缺失步骤

### 11. YAGNI 与反模式 [通用]

**判断标准**：
- 是否过度复杂？
- 是否滥用 MUST/ALWAYS/NEVER？
- 是否有不必要的抽象层？

**Pass**: 简洁实用，无冗余
**Warning**: 略有过度设计
**Issue**: 明显过度复杂或充斥强制性指令

## 评分映射

将 11 个维度的评估结果映射为内容质量分数：

| 结果分布 | 分数范围 |
|----------|----------|
| 全 Pass | 95-100 |
| ≤ 2 Warning | 80-94 |
| 3+ Warning 或 1 Issue | 60-79 |
| 2+ Issue | 30-59 |
| 5+ Issue | 0-29 |
